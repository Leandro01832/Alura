Referente ao curso ASP.NET Core Parte 4: IdentityServer, no capítulo Criando e Executando Projeto IdentityServer4 e atividade Criando Novo Projeto Identity Server 4

27/12/2019
LINDALVO GONZAGA DE ALCANTARA NETO
(159.4k xp, 3 posts)
Analista Judiciário

Toda vez que tento criar o projeto aparece o erro: Ocorreu uma falha no build. Corrija os erros de build e execute novamente. Ao tentar rodar o dotnet run /seed aparece o erro:

C:\Program Files\dotnet\sdk\2.2.204\Sdks\Microsoft.NET.Sdk\targets\Microsoft.NET.TargetFrameworkInference.targets(137,5): error NETSDK1045: O SDK do .NET atual não dá suporte para direcionar a .NET Core 3.0. Direcione a .NET Core 2.2 ou inferior, ou use uma versão do SDK do .NET compatível com .NET Core 3.0. [C:\Users...\CasaDoCodigo.IdentityServer\CasaDoCodigo.IdentityServer.csproj]

O Visual Studio 2017 não consegue compilar o projeto importado. Descobri que o comando dotnet new está gerando um projeto com a tag netcoreapp3.0 apesar de não ter tal SDK instalado

C:\Users...\IdentityServer\CasaDoCodigo.IdentityServer>dotnet --list-sdks 2.1.509 [C:\Program Files\dotnet\sdk] 2.2.204 [C:\Program Files\dotnet\sdk]

Versão do Visual Studio 2017 = 15.9.18

Já fiz várias pesquisas na internet, já tentei criar o arquivo global.json e cheguei até mesmo editar manualmente o arquivo do projeto direcionando para o SDK 2.2.204. Nada funcionou. Alguém mais está com este problema ?
