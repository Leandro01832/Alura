[00:00] A partir de agora, vamos começar a trabalhar com uma nova pasta na solução, que é a item03 criando e executando o projeto identity. Temos dois projetos que foram copiados da pasta item02, o MVC e o relatório web API. Só temos que garantir que eles são os projetos iniciais da solução, clicando com o botão direito e escolhendo o definir projeto de inicialização, garantindo que os dois projetos estejam com start.

[00:46] Feito isso, sabemos que o identity server 4 vai permitir a autenticação de usuários tanto na aplicação MVC como no serviço web API. E já conhecemos alguma características do identity server. Iremos começar criando um novo projeto identity server 4 na solução.

[01:20] Primeiro vamos abrir o utilitário de linha de comando do visual studio, que é o developer command prompt. Nele, vamos navegar para a pasta da sua solução. Temos que ter o diretório onde está localizada a solução. Mudo o diretório atual usando o change directory. Mas tenho que fazer isso entre aspas porque pode ter espaços em branco no caminho que eu copiei.

[03:20] Ainda preciso navegar para a pasta item03. Vamos fazer novamente um change directory. Agora, vamos ter que criar uma nova pasta nesse diretório com o nome do projeto que vamos criar. Vamos colocar o comando para isso, que é o make directory e o nome do projeto. Também preciso navegar para essa pasta.

[05:06] Estamos nessa pasta nova e vamos fazer a instalação de um novo projeto usando linha de comando, mas preciso primeiro fazer a instalação de templates de projetos do identity server. Para isso, usamos o comando dotnet new, que vai nos permitir criar um novo projeto definido pelo template que vou informar. Coloco a opção -i para fazer a instalação de templates, e depois coloco o nome dos templates.

[06:42] Temos que rodar um comando para fazer a instalação de um dos templates disponíveis nessa instalação. Nós iremos usar o is4aspid. Ele vai fazer a instalação de um novo projeto com o Aspnet core identity. Posso copiar o código e na linha de comando digito a instrução para criar um novo projeto com o dotnet new.

[07:58] Ele pede uma confirmação para que eu faça o preenchimento do campo de dados com usuários de exemplo. Vou executar para ganhar dois usuários cadastrados na base para podermos testar.

[09:20] O próximo passo é colocar o comando para listar o diretório. Ele gera os arquivos do novo template. Vamos adicionar esse novo projeto à nossa solução. No visual studio, na pasta do item03 vou clicar com o botão direito e adicionar o projeto existente. Seleciono a pasta com o novo projeto e seleciono o projeto criado.

[10:27] Vamos ter que fazer algumas mudanças. Entre elas, definir o projeto como um projeto de inicialização. Clico com o botão direito sobre a solução e escolho set startup projects para marcar o novo projeto também como projeto de inicialização.

[11:00] Para organizar melhor os projetos nessa lista vou mover a linha para o final. Ele vai ficar com os outros projetos da pasta. Para isso, uso a seta para baixo. Outra mudança que vamos fazer é alterar a porta da aplicação. Iremos clicar com o botão direito, ir em propriedades e escolher a aba debug. Ele definiu a porta como 5000, é o que queremos. Se você criou alguma com porta diferente, mude para 5000.

[12:42] Temos os três projetos sendo executados. A identity server 4 permite que você faça o login através dela, clicando no link. Aqui iremos fazer o login através dos usuários disponíveis. São eles a Alice e o Bob. Ambos tem a mesma senha.

[14:52] Se ela quiser, a Alice pode fazer o logout diretamente nessa aplicação. Ela vai clicar no seu nome e vai aparecer a opção de menu de logout. Clicando no link, ele pede uma confirmação, clico no sim e o usuário está desconectado com sucesso.

[15:25] Nesta etapa do curso entendemos as necessidades de um novo serviço de autenticação de usuário centralizada numa aplicação, arquivo de serviços. Usamos um serviço de token de segurança com um novo projeto identity server 4 como centralizador do serviço de autenticação para nossa aplicação web de comércio eletrônico. Nas próximas etapas vamos ver como integrar esse projeto identity server 4 com a aplicação MVC para que ela possa usufruir dos benefícios da autenticação de usuários.
